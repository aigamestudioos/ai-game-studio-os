# AGENT.md

Guia de referência para agentes (humanos ou IA) que trabalham neste repositório.

## Ordem de Leitura Obrigatória

Antes de qualquer implementação, leia nesta ordem:

1. PROJECT_STATUS.md
2. ARCHITECTURE.md
3. DECISIONS.md
4. Documentos relevantes em docs/
5. Somente então iniciar qualquer alteração.

## Regras Permanentes

- Nunca assumir requisitos ausentes.
- Nunca alterar arquitetura sem autorização.
- Nunca criar dependências fora das SPECs.
- Sempre inspecionar antes de modificar.
- Sempre validar antes de commitar.
- Commits pequenos e atômicos.
- O repositório é a única fonte oficial da verdade.
