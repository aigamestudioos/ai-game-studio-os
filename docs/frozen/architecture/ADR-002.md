# ADR-002 — Arquitetura Oficial do Monorepo

**Status:** APPROVED FROZEN
**Data:** 14/07/2026

---

## Decisão

O AI Game Studio OS utilizará **pnpm workspaces + Turborepo** em repositório único GitHub.

## Estrutura Oficial

```
ai-game-studio-os/
├── apps/web/
│   ├── app/
│   ├── features/     ← NUNCA features/ na raiz do repositório
│   ├── components/
│   ├── providers/
│   ├── hooks/
│   ├── lib/
│   └── tests/
├── packages/
│   ├── ui/           @agsos/ui
│   ├── database/     @agsos/database
│   ├── auth/         @agsos/auth
│   ├── events/       @agsos/events
│   ├── config/       @agsos/config
│   ├── validation/   @agsos/validation
│   ├── observability/@agsos/observability
│   ├── integrations/ @agsos/integrations
│   ├── storage/      @agsos/storage
│   ├── testing/      @agsos/testing
│   └── i18n/         @agsos/i18n
├── supabase/
├── docs/frozen/
├── scripts/
└── .github/workflows/
```

## Direção de Dependências

```
apps/web → features locais → packages compartilhados
```

- Packages **nunca** importam de apps/web ou de outras features
- Features **nunca** importam arquivos internos de outras features
- Toda API pública via `index.ts`

## Critério para Criar Package

Mover para packages/ somente quando: usado por mais de uma aplicação, infraestrutura transversal, API pública claramente definida, testável isoladamente.

## @agsos/types

**Não existe.** Tipos distribuídos por: `packages/database` (banco), features (domínio), package proprietário (compartilhados).

## Turborepo (turbo.json)

Pipelines: build, dev, lint, typecheck, test, test:e2e, format, format:check, clean.

- build: depende do build das dependências internas
- dev: persistente, sem cache
- lint/typecheck/test: executáveis da raiz com cache

## pnpm

- `pnpm-workspace.yaml`: `apps/*` + `packages/*`
- Dependências internas: `workspace:*`
- Não importar arquivos internos de outro package
- `packageManager` fixado no `package.json` raiz
