# ADR-003 — Estratégia Oficial de Acesso a Dados, Supabase e RLS

**Status:** APPROVED FROZEN
**Data:** 14/07/2026

---

## Decisão

O AI Game Studio OS utilizará **Supabase nativo** como única camada de dados.

**Prisma foi removido da stack.**

## Stack de Dados

- Supabase PostgreSQL
- Supabase CLI
- Migrations SQL versionadas (fonte oficial do schema)
- Supabase Auth
- Supabase Client (supabase-js)
- Tipos TypeScript gerados (`supabase gen types typescript`)
- Row Level Security (RLS)
- PostgreSQL Functions / RPCs
- Supabase Edge Functions (runtime: Deno)
- Supabase Storage
- Supabase Realtime (restrito)

## Três Clientes Separados

```typescript
// packages/database/src/
browser-client.ts   // Client Components — apenas chave pública, sujeito a RLS
server-client.ts    // Server Components / Server Actions — contexto de sessão, sujeito a RLS
admin-client.ts     // Operações admin — service role, APENAS servidor, auditado
```

## RLS

- Habilitado em **todas** as tabelas de negócio sem exceção
- Toda tabela de domínio possui `studio_id`
- Isolamento garantido pelo banco, nunca apenas por filtros de frontend
- Admin client: cada operação que ignora RLS deve ter auditoria e justificativa

## Tipos TypeScript

```bash
supabase gen types typescript --project-id REF > packages/database/src/generated/database.types.ts
```

**Nunca editar manualmente `database.types.ts`.**

## Migrations

- **Forward-only** — nunca criar `down.sql`
- Uma migration = um contexto de negócio
- Mudanças destrutivas: estratégia expand-and-contract
- Fluxo: local → staging → produção

## Condição para Reavaliar Prisma

Prisma somente reconsiderado se surgir necessidade concreta não atendida pela arquitetura Supabase nativa. Exige: novo ADR, análise de segurança, estratégia de autorização, prova de benefício superior à complexidade adicionada.

## Estrutura

```
supabase/
├── migrations/    ← fonte oficial do schema
├── seed/
├── functions/     ← Edge Functions (Deno)
├── tests/         ← testes de RLS
└── config.toml

packages/database/src/
├── browser-client.ts
├── server-client.ts
├── admin-client.ts
├── generated/
│   └── database.types.ts   ← gerado automaticamente
├── repositories/
├── queries/
└── mutations/
```
