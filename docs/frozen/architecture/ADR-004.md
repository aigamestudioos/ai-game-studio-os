# ADR-004 — Client State & Server State

**Status:** APPROVED FROZEN
**Data:** 14/07/2026

---

## Decisão

| Estado | Tecnologia |
|---|---|
| Server State (dados remotos) | TanStack Query |
| Client/UI State (interface) | Zustand |

## TanStack Query

Responsável por: dados remotos, cache, invalidação, sincronização, re-fetching automático.

Usado em: queries de leitura, listas, detalhes, dados paginados.

**Nunca** armazenar dados persistentes do negócio em Zustand.

## Zustand

Responsável por: estado temporário da interface, preferências do usuário na sessão, UI state (painéis abertos, seleções temporárias, modo de visualização).

## Justificativa

TanStack Query não estava na stack da SPEC-001 original. Adicionado via ADR-004 pois: resolve o problema de gerenciamento de server state sem boilerplate excessivo, é padrão no ecossistema Next.js App Router, substitui soluções ad-hoc que surgiriam naturalmente.

## Integração com App Router

- Server Components: dados via `async/await` direto (sem TanStack Query)
- Client Components: TanStack Query para dados reativos e atualizações
- Server Actions: invalidação de queries após mutações bem-sucedidas
