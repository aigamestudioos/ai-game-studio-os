# UL-001 — Ubiquitous Language

**Versão:** 1.0.0
**Status:** APPROVED FROZEN
**Classificação:** Documento Normativo

---

## Objetivo

Estabelecer o vocabulário oficial do AI Game Studio OS.

Todos os documentos, código, banco de dados, APIs e interfaces deverão utilizar exclusivamente os termos definidos neste documento.

---

## Termos Oficiais

### 1. Studio
Organização proprietária do AI Game Studio OS.
Não confundir com: Workspace.

### 2. Workspace
Ambiente de trabalho dentro do sistema.
Um Studio possui um Workspace.

### 3. User
Pessoa autenticada que utiliza o sistema.

### 4. Role
Função atribuída a um usuário.
Ex.: Owner, Admin, Developer.

### 5. Permission
Autorização específica concedida a uma Role.

### 6. Idea
Hipótese de um novo jogo ou funcionalidade.
Pode virar Project.

### 7. Project
Unidade de trabalho criada para transformar uma Idea em um produto.
Pode gerar um ou mais Games.

### 8. Epic
Grande bloco de trabalho dentro de um Project.

### 9. Feature
Capacidade funcional do sistema ou de um jogo.
**Não confundir com** o diretório técnico `apps/web/features/` — são contextos distintos.

### 10. Task
Menor unidade planejável de trabalho.

### 11. Milestone
Marco importante de um Project.

### 12. Game
Produto digital desenvolvido pelo Studio.

### 13. Version
Versão lógica de um Game. Ex.: 1.2.0.

### 14. Build
Artefato compilado de uma Version.

### 15. Release
Versão publicável de um Game. Pertence ao domínio Games.
Publishing consome Releases; não as possui.

### 16. Platform
Destino da publicação. Ex.: Apple App Store, Google Play.

### 17. Campaign
Conjunto de ações de marketing. Aggregate Root do domínio Marketing.

### 18. Creative
Material gráfico ou audiovisual utilizado em uma Campaign.

### 19. Asset
Arquivo utilizado por um Game ou Campaign.

### 20. Prompt
Instrução enviada a uma IA.

### 21. Prompt Template
Modelo reutilizável de Prompt.

### 22. AI Agent
Especialização de IA responsável por uma função específica.

### 23. AI Orchestrator
Serviço responsável por coordenar tarefas executadas por IA.
Não possui tabela própria — é camada de orquestração sobre `jobs`, `prompt_executions`, `studio_events`, `conversations`.

### 24. Automation
Fluxo executado automaticamente pelo sistema.

### 25. Event
Registro imutável de algo relevante ocorrido no sistema.

### 26. Notification
Mensagem apresentada ao usuário. Pertence ao domínio Infrastructure.

### 27. Specification (SPEC)
Documento técnico que define como algo deve ser implementado.

### 28. ADR
Architecture Decision Record. Documento que registra uma decisão arquitetural permanente.

### 29. SOP
Procedimento operacional padrão. Define como executar uma atividade.

### 30. Playbook
Guia estratégico para execução de um processo recorrente.

### 31. Integration
Conexão com um serviço externo.

### 32. Environment
Ambiente de execução. Ex.: Local, Development, Staging, Production.
Pertence exclusivamente ao domínio Infrastructure. Studio referencia apenas `active_environment_id`.

### 33. Deployment
Processo de publicação do sistema.

### 34. Knowledge Entry
Informação estruturada armazenada na base de conhecimento.

### 35. Domain
Área de negócio independente dentro do sistema.
Ex.: Studio, Projects, Games, Finance.

---

## Regras

- Um termo possui apenas um significado oficial.
- Sinônimos não são permitidos na documentação técnica.
- Nomes de tabelas, APIs e código devem seguir este vocabulário.
- Novos termos exigem atualização deste documento antes de uso.
